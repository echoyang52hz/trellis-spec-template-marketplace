## 修改前搜索规则

在修改任何值、字段、配置、常量、路径、状态名或约定前，先搜索已有使用点。

推荐优先使用 `rg`：

```bash
rg -n "value_to_change"
```

如果环境中没有 `rg`，使用项目可用的搜索工具替代。

这个习惯可以避免只改一个位置，却漏掉调用方、文档、测试、配置或历史兼容逻辑。
